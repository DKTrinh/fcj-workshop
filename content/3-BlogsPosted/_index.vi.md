### [Blog 1 - SESSION POLICIES TRONG AMAZON EKS POD IDENTITY](3.1-Blog1/)
Blog này giới thiệu Amazon EKS Pod Identity vừa bổ sung tính năng session policies, cho phép bạn thu hẹp quyền IAM một cách linh hoạt và chính xác cho từng pod mà không cần tạo thêm nhiều IAM roles riêng biệt. Đây là bước tiến quan trọng giúp áp dụng nguyên tắc least privilege hiệu quả hơn trong môi trường Kubernetes quy mô lớn.

### [Blog 2 - TRIỂN KHAI ỨNG DỤNG DOCKER LÊN AWS ECS VỚI FARGATE](3.2-Blog2/)
Blog này hướng dẫn cách đóng gói ứng dụng web thành Docker image và triển khai tự động lên AWS thông qua Amazon ECS kết hợp với AWS Fargate. Kiến trúc Serverless này giúp lập trình viên không cần lo lắng về việc quản lý hay bảo trì hạ tầng máy chủ ảo (EC2), từ đó tối ưu hóa thời gian và chi phí vận hành dự án.

### [Blog 3 - LƯU TRỮ VÀ PHÂN PHỐI ỨNG DỤNG REACT VỚI AWS S3 VÀ CLOUDFRONT](3.3-Blog3/)
Blog này phân tích giải pháp tối ưu để host một ứng dụng Single Page Application (như React) trên AWS. Bằng cách kết hợp tính năng Static Website Hosting của Amazon S3 và mạng phân phối nội dung (CDN) Amazon CloudFront, ứng dụng có thể đạt tốc độ tải trang cực nhanh trên toàn cầu, đồng thời được bảo mật SSL chuẩn quốc tế.
